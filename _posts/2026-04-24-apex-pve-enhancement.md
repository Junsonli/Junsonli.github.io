# 1. 玩法概述

玩法体验：**娱乐向玩法**，在大逃杀对局基础上，给予玩家随机强化（3选1），增强趣味性以及丰富战斗体验

玩法模式：以BR对局为基础，缩圈规则、关卡地图与常规BR对局完全一致。

（关卡上可以考虑增加一些类似爆炸舱室的小POI，可以升级强化）

# 3. 局内规则

## 3.1 模式赛制

规则与常规大逃杀对局一致。

* 例如：关卡、获胜条件、刷圈规律、制造器机制等主要机制均保持一致

## 3.2 基本规则简介

玩家共有4个强化槽位，在达到结算节点统一解锁1个强化

* 强化解锁由玩家进行3选1
* 解锁不拍脸，不打断玩家操作，类APEX进化系统进行提示，玩家可在节点后随时选择强化

## 3.3 强化解锁

### 3.3.1 强化解锁节点

Apex大逃杀对局节奏：

| 回合数                        | 安全区半径（m） | 回合时间 | 结束后缩圈时间 |
| -------------------------- | -------- | ---- | ------- |
| 1（**跳伞、落地阶段**）             | 无        | 60s  | 270s    |
| 2（**第一个圈**）                | 1000     | 165s | 45s     |
| 3                          | 650      | 135s | 45s     |
| 4                          | 400      | 105s | 40s     |
| 5                          | 200      | 90s  | 30s     |
| 6<br>（**决赛圈，结束后安全区完全闭合**） | 100      | 60s  | 120s    |

**解锁节点：**
**方案1 **—— 随回合数发放
* 第1个强化：游戏开局发放，玩家在运输船上选择强化解锁
* 第2个强化：第2回合开始时为解锁节点
* 第3个强化：第3回合开始时为解锁节点
* 第4个强化：第4回合开始时为解锁节点
**方案2** —— 随存活人数发放（3人赛）
* 第1个强化：游戏开局发放，玩家在运输船上选择强化解锁
* 第2个强化：当剩余存活人数为45人发放
* 第3个强化：当剩余存活人数为30人发放
* 第4个强化：当剩余存活人数为18人发放
**目的：**让半数以上玩家能够体验到2\~3个强化的体验，近决赛圈以及决赛圈为4强化体验。根据游戏数据做选择。

### 3.3.2 解锁与使用强化表现包装

1. 解锁提示：长红方框，玩家按【】键进入选择界面，选择界面为轮盘
2. 使用：按【】键使用主动强化，同为轮盘
3. 玩家解锁与使用强化表现：
    * 解锁的动画基于绝招加速器动画作修改
        * 强化解锁无读条等待，玩家选取完毕后，衔接纯动画，不严重影响玩家操作（同APEX滑索，可变滑索变开枪）
    * 玩家获取的强化在右侧显示简图
        * 简图需要分为多种状态：未激活态、激活态、冷却态、限时生效倒计时态

## 3.4 拾取与替换机制&死亡惩罚

1. **玩家可以通过敌人的死亡之箱拾取强化**
    * 拾取的强化只能替换掉已有的强化机制，不可新加（保证全局存活玩家都为
        * 被替换的强化会消失（类似于杀戮尖塔的消耗烧牌）
            * 消失的强化，本局3选1不再刷新，但仍可以从 <u>敌人死亡之箱</u> 以及 <u>重生后的降级随机</u> 获取
    * 每个玩家只能从一个死亡之箱中替换一个强化

     > A玩家的死亡之箱有3个强化，B玩家只可以从中获取1个，需要3个玩家替换才可以将A的强化全部替换

2. **玩家替换强化需要时间，不可以秒替换**
    * 替换动画参考正常使用绝招加速器，但手机屏内显示需要新资源
    * 替换时可以随时按【Z】键取消替换
    * 在一人（玩家A）开始替换到完全替换期间，死亡之箱内的被替换的强化会呈不可选态（灰色暗淡），其他人（玩家B）无法在此期间选择该强化替换，若前者（玩家A）取消替换后可选中开始替换
3. **死亡惩罚（强化被替换惩罚）**
    * 玩家死亡之箱的强化被替换后，将会使用随机的更低级强化代替其被他人拾取的强化，没有3选1机会
        * 新的随机强化等级不会高于被替换强化（A级→B级→C级）

     > 例如玩家A死亡后到复活前的时间，其N个强化均被瓜分，其重生将拥有新的N个更低等级的随机强化，不给予其3选1机会。

# 5. 设计思路相关

基本的设计原则：

让每张强化都拥有生态位

大概的设计方法论：

对于不同维度的强化设计，先细分该维度可以涉及的细分模块都有什么，理清楚可以涉及的模块

参考优质游戏、同类产品、其他品类产品的设计 + 玩家行为代入 + 广泛传播的有趣梗图或者视频 进行设计

设计输出，同时对设计进行边缘情况思考，避免有重大遗漏（与人同步以及讨论效率就很高，接受质问）

以3C能力为例子，设计思路：

1. 思考能对3C改动的地方主要为移动与视角，增加某种机制或者加强某种移动（跳得更高，滑得更远），对视角进行变化等
